## 一、配图不足：可行方案

### 1. 现有资源复用（零成本）

- **已有 6 张图**：hero、product_group、camera_12mp、hardware_arch、look_and_ask、industry_chain。
- **做法**：在目前无图的章节里，从已有图里「借」一张做小图/背景，强化视觉。
  - 例：**设计创新** 可放一小块 product_group 或 hardware_arch 做插图；**应用场景** 用 look_and_ask 或 product_group 做顶图或侧边图。
- **优点**：不改动素材，只改布局。**缺点**：同一张图出现多次，信息量不增加。

### 2. 用占位图 + 后续替换（先有版式再补图）

- **做法**：在无图章节加入固定尺寸的「图位」：用占位色块 + 简短文案（如「产品多角度图」「市场数据图」），版式先定好。
- **后续**：你从 Meta 官网、报告、评测里存图到 `src/assets/`，再替换占位图的 `src` 即可。
- **优点**：立刻让每章都有「图区」，视觉更均衡；补图时只换资源不改结构。**缺点**：占位阶段略抽象，需要你后续花时间找图。

### 3. 用数据/信息图代替照片（无需新照片）

- **做法**：对**技术参数、市场反响、竞品对比、痛点**等章节，用「图表」当配图，而不是实物图：
  - **技术参数**：简单条形图/雷达图（如重量、续航、价格对比），可用 Recharts（项目里已有）或静态 SVG。
  - **市场反响**：销量/市占的小型柱状图或数字大号展示。
  - **竞品对比**：表格已有，可加一栏「缩略图」或小型对比卡片配图标/色块。
  - **痛点**：用 3～4 个图标 + 一句话的卡片排布，视觉上像「图」。
- **优点**：不依赖新照片，和课程汇报的数据风格一致。**缺点**：需要一点图表配置或 SVG 编写。

### 4. 明确建议补图的章节与类型（你找图，我接进页面）

- 建议**优先补图**的章节与类型：
  - **产品构成**：产品多角度/爆炸图（若有），或镜腿特写。
  - **应用场景**：1 张「使用场景」照片或合成图（如佩戴 + 手机/翻译界面）。
  - **市场反响**：机构报告里的出货/市占图（截图或重绘均可）。
  - **竞品对比**：各品牌产品小图一排，或 1 张对比图。
  - **小结**：可选 1 张有「未来感」的图或产品图收尾。
- 你只要把图存到 `src/assets/` 并告诉我文件名，我可以帮你写好在对应章节的 `<Figure>` 或 `<img>` 位置。

## 二、章节分界不明显：可行方案

### 1. 加强「章节标题区」的视觉重量（推荐，改动小）

- **做法**：每个章节的「标题区」从「编号 + 标签 + 一根线」升级为更显眼的一块：
  - 左侧或上方加一条**粗一点的竖线/色条**（用品牌色，如青色/紫），或小色块。
  - 编号放大、字重加大，或给编号加圆角矩形背景。
  - 可选：章节标题（如「案例选择说明」）用稍大字号或不同字重。
- **效果**：不改变整体布局，只让「每一章的开头」更醒目，滚动时一眼能看出新章节。

### 2. 给每个章节加「卡片容器」背景（分界最明显）

- **做法**：每个 `<section>` 包在一个带背景的容器里，例如：
  - 浅色底（如 `bg-white/5`）+ 圆角 + 内边距，或
  - 细边框（如 `border border-white/10`）+ 内边距。
- **效果**：每章是一块「卡片」，章节之间自然留白 + 背景差异，分界非常清晰。
- **注意**：若每章都加底，整体会略「重」；可只对部分重点章节（如 01、05、09、12）用卡片，其余保持现状。

### 3. 章节之间加大间距 + 分隔线

- **做法**：
  - 章节间距从 `space-y-16` 改为 `space-y-20` 或 `space-y-24`。
  - 在每两个 section 之间加一条**横线**（或渐变线），线宽 1px，颜色与现有 Hairline 一致或略深。
- **效果**：不用改章节内部结构，只靠「留白 + 线」强化分界。实现简单，风格保持简洁。

### 4. 每章开头加「章节大号」装饰（适合强调节奏）

- **做法**：在每章左上角或左侧放一个很大的**章节编号**（如 01、02…）作为装饰，弱化颜色（如 `text-white/10` 或 `text-white/15`），字很大（如 6rem～8rem），部分被内容遮挡。
- **效果**：类似杂志/年报的「章节大码」，既有分界感又不抢正文，和现有极简风格兼容。

### 5. 组合方案（分界 + 配图一起用）

- 例如：**章节用「卡片容器」+ 章节内增加「图位/数据图」**：
  - 每章一块卡片，卡片内上半部分为「标题 + 一小段文字」，下半部分为「一张图或一组图表」。
  - 这样「分界」和「配图」同时加强，版式也更统一。

## 三、建议的优先顺序（供讨论）

1. **先做章节分界**（成本低、收益直观）  
   - 优先试：**方案二·1**（加强章节标题区）+ **方案二·3**（加大间距 + 分隔线）。  
   - 若希望分界更强，再考虑 **方案二·2**（部分章节加卡片容器）。

2. **再补配图**  
   - 短期：**方案一·2**（无图章节先加占位图位，再逐步替换为真实图或数据图）。  
   - 中期：按 **方案一·4** 在你方便时补 3～5 张图，我帮你接到对应章节。  
   - 若你更倾向「用数据说话」：在参数/市场/竞品/痛点用 **方案一·3**（图表/卡片当图）。

3. **可选**  
   - **方案二·4**（大号章节编号）适合想加强「汇报节奏感」时再加。  
   - **方案一·1**（现有图复用）适合某几章临时需要一点图、又暂时不想找新图时用。
